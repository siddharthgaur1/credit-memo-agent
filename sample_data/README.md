# sample_data — SYNTHETIC ONLY

Every file produced here is **invented**. The borrowers, PANs, GSTINs, account
numbers, addresses and figures are fictional and were generated by
`generate.py`. Nothing in this directory came from a real customer, a real bank
file, or any real credit application.

This exists so the project can be demonstrated — including in `LLM_BACKEND=openai`
mode, where document content is sent to an external API — without any real
borrower data being involved.

**Do not place real borrower documents in this directory.** They belong under
`data/local/`, which is gitignored.

## Generate

```bash
python sample_data/generate.py            # writes sample_data/out/<borrower>/
```

Three files are produced:

| Borrower | What it demonstrates |
|---|---|
| `suryodaya_engineering` | Healthy: complete file, clean ratios, no flags of substance |
| `vaishnavi_polymers` | Stressed: revenue decline, DSCR under 1, negative PAT, cheque returns |
| `meenakshi_traders` | Incomplete: no GST returns, financials 4+ years stale, a balance sheet that **deliberately does not balance** and a P&L with depreciation missing |

`meenakshi_traders` is the honesty test: the tool must report the broken balance
sheet as a finding and report DSCR as *insufficient data* rather than inventing
the missing depreciation figure.
